## Key Learnings

- **Project:** banlieue
- **Description:** A Kubernetes-native abstract virtualization API. Users create `VirtualMachine` CRs; controllers schedule them onto vSphere / Proxmox / libvirt backends via provider-specific infrastructure CRDs that satisfy the CAPI v1beta2 InfraMachine contract.
- **Architecture invariant:** No RPC between main controller and providers â€” communication is **CRD-only**. If a design wants HTTP/gRPC between them, it's wrong.
- **Workspace layout:** Cargo workspace. Phase 0 (done) ships `crates/banlieue-api` with two API groups: `banlieue.io/v1alpha1` (`Provider`, `VMClass`, `VMImage`, `VirtualMachine`) and `infrastructure.banlieue.io/v1alpha1` (`VSphereMachine`, `VSphereMachineTemplate`). Phase 1+ adds `banlieue-controller`, `banlieue-provider-sdk`, and `banlieue-provider-{vsphere,proxmox,libvirt}` per the out-of-repo roadmap at `~/dev/roadmaps/banlieue/`.
- **CRD generation:** `cargo run -p banlieue-api --bin crdgen --features crdgen` â€” the `crdgen` binary is feature-gated, so the flag is required. `serde_yaml` is also gated on the `crdgen` feature via `crdgen = ["dep:serde_yaml"]`.
- **License:** Apache-2.0 (workspace-wide, set in `[workspace.package]`).
- **Roadmap filename convention:** UPPERCASE with hyphens, numeric prefix (`00-OVERVIEW.md`, `10-PHASE-1A-CONTROLLER-AND-SDK.md`). ADRs use lowercase-hyphen.
- **k8s-openapi needs the `schemars` feature** for `Condition`, `Time`, and other meta/v1 types to implement `JsonSchema`. Without it, any field like `Vec<Condition>` or `Option<Time>` in a `JsonSchema`-deriving struct fails to compile. Pinned in workspace `[workspace.dependencies]`.
- **Test file colocation requires `#[path]`** because `src/foo.rs` is a leaf file-module. Pattern: at the bottom of `src/foo.rs` put `#[cfg(test)] #[path = "foo_tests.rs"] mod foo_tests;`, then put a `mod tests { use super::super::*; ... }` inside `src/foo_tests.rs`. Without `#[path]`, rustc looks for `src/foo/foo_tests.rs`. Test files do **not** need to re-import `crate::common::*` â€” `use super::super::*` already pulls in everything in scope in the parent module (child modules see private `use` imports from the parent).
- **kube-derive's CRD flattener cannot handle serde-tagged enums** (the actual root cause behind the original buglog bug-006). Each variant subschema of a `#[serde(tag = "x")]` enum naturally fixes `x` to a different enum value, but kube-core demands identical schemas for properties shared across subschemas. The CRD-generation panic is therefore fundamental to tagged enums, not a metadata bug. **The correct Kubernetes pattern is a flat struct** with a discriminator-string field plus optional sibling fields per variant. This is how `IpamSpec` was redesigned on 2026-05-25 (bug-006 fixed). Future learning: do not introduce `#[serde(tag = ...)]` enums on any type that needs to land in a CRD.
- **k8s-openapi 0.27 uses `jiff::Timestamp` for `Time`, not `chrono`.** Code that referenced `k8s_openapi::chrono::DateTime<Utc>` fails to compile. To produce a `Time` in tests without a direct jiff dep, deserialize an RFC3339 string: `serde_json::from_str::<Time>(&format!("\"{}\"", rfc3339)).unwrap()` â€” this works because `Time` is `#[derive(Deserialize)]`.
- **kube-rs reference layout (kube-rs/controller-rs):** single-crate, with `src/lib.rs` (Error enum + re-exports), `src/main.rs` (actix-web + tokio), `src/controller.rs` (CustomResource + reconciler + Context + finalizer + events), `src/crdgen.rs`, `src/metrics.rs`, `src/telemetry.rs`, `src/fixtures.rs`. Root-level: `Dockerfile`, `Tiltfile`, `justfile`, `charts/`, `yaml/`. **banlieue is multi-crate by design** (controller + provider SDK + 3 providers) so we adopt their *per-crate conventions*, not their single-crate layout.
- **Pinned dep versions (2026-05-25, matching controller-rs):** `kube = "3"` (features `derive`, `client`, `runtime`), `k8s-openapi = "0.27"` (features `latest`, `schemars`), `schemars = "1"`, `thiserror = "2"`, `tokio = "1"`, `tracing-subscriber = "0.3"`, `futures = "0.3"`, `anyhow = "1"`. Edition `2024`, MSRV `1.85`. Added 2026-05-26 in Phase 1A: `clap = "4"` (features `derive`, `env`), `chrono = "0.4"` (features `serde`), `async-trait = "0.1"`; `tracing-subscriber` gained the `json` feature.
- **Controller crate needs extra tokio features.** `crates/banlieue-controller/Cargo.toml` pins `tokio = { workspace = true, features = ["macros", "rt-multi-thread", "net", "io-util", "signal"] }`. The bare workspace `tokio` only ships `macros + rt-multi-thread`, which is fine for the SDK lib but not for `main.rs`'s health TCP listener.
- **5-spot-style Docker pattern (adopted 2026-05-26):** single `Dockerfile` and `Dockerfile.chainguard` at workspace root, both parameterised by a `BINARY` build-arg, so every banlieue binary (controller + every future provider) ships from the same two Dockerfiles. They never compile inside the container â€” the Makefile cross-compiles to `target/<triple>/release/<bin>`, stages at `binaries/<arch>/<bin>`, and the Dockerfile `COPY`s it in. Bases pinned by digest (distroless cc-debian13:nonroot and Chainguard glibc-dynamic). EXPOSE 8080 (metrics) and 8081 (health). User: `nonroot` (distroless) or `65532:65532` (Chainguard).
- **5-spot-style Makefile pattern (adopted 2026-05-26):** all workflow logic lives in the Makefile, not in GH-Action YAML. Targets: `help` (parses `## comments`), `build`/`test`/`lint`/`format`/`clean`, `crds` (runs `cargo run -p banlieue-api --bin crdgen --features crdgen -- --out-dir deploy/crds`), `run-local` (cargo run controller against current kube-context), `kind-{install,create,delete,up,deploy-crds,deploy-controller,load,status}` and `docker-{build,buildx,build-chainguard,buildx-chainguard}` (parameterised by `BINARY=<name>` so multi-binary workspace scales). `make kind-up` is the ideal local dev entry: creates the kind cluster + applies CRDs; controller runs out-of-cluster via `make run-local`.
- **Local-dev kind image override pattern:** `deploy/controller/deployment.yaml` pins `image: ghcr.io/firestoned/banlieue-controller:v0.1.0` (NEVER `:latest`); the Makefile's `kind-deploy-controller` target uses `kubectl set image deployment/banlieue-controller controller=$(KIND_IMAGE)` to swap in the locally-built `ghcr.io/firestoned/banlieue-controller:local-dev` image after applying manifests. Matches 5-spot's `kind-deploy` target.
- **`crdgen` supports two output modes (2026-05-26):** default stdout multi-document stream (for `kubectl apply -f -` piping); `--out-dir <DIR>` writes one file per CRD using the kubebuilder convention `<group>_<plural>.yaml`. The Makefile `crds` target uses `--out-dir deploy/crds`.
- **SDK module map (2026-05-26):** `banlieue-provider-sdk` exposes `client::build_client` (env + kubeconfig + timeouts), `status::{set_condition, is_condition_true, find_condition}` (sorted-by-type, transition-time-preserving), `finalizer::{ensure_finalizer, remove_finalizer}` plus pure helpers `finalizer_list_with`/`finalizer_list_without` (testable without a kube client), `ssa::{server_side_apply, FIELD_MANAGER_*}` (one constant per controller: `banlieue.io/controller`, `banlieue.io/provider-vsphere`, etc.), and `reconciler::{requeue_default(30s), requeue_on_error(5s), requeue_long(300s), no_requeue}`.
- **Time construction without chrono dep (2026-05-26 reinforcement):** to construct `k8s_openapi::apimachinery::pkg::apis::meta::v1::Time` directly in code (production or tests), use `k8s_openapi::jiff::Timestamp::now()` â€” k8s-openapi re-exports `jiff` at its root so we don't need an extra dep. `chrono::Utc::now()` does NOT work because `Time` wraps `jiff::Timestamp` in 0.27.
- **SDK error enum must include all kube config error variants.** `kube::Config::infer().await` returns `InferConfigError`, NOT `KubeconfigError` or `InClusterError`. Forgetting to add the `#[from] InferConfigError` variant breaks the `?` in `client::build_client`. Variants currently in `banlieue_provider_sdk::Error`: `Kube`, `KubeConfig` (KubeconfigError), `InClusterConfig` (InClusterError), `InferConfig` (InferConfigError), `Serde`, `Missing(&'static str)`.
- **Scheduler shape (Phase 1A iter 2, 2026-05-26):** pure function `crates/banlieue-controller/src/reconciler/scheduler::schedule(vm, class, image, providers, existing_vms) â†’ Result<Decision, ScheduleError>`. Owned `Decision` carries `provider_name`, `provider_namespace`, `provider_class`, `failure_domain_name`, `resolved_storage` / `resolved_networks` (both `Vec<ResolvedResource>`), `failure_domain_raw` (BTreeMap from `attributes.raw` â€” opaque, consumed by provider-specific infra builders), and `failure_domain_labels`. Filter chain order matters: providerSelector â†’ failureDomainSelector â†’ image readiness â†’ storage classes â†’ network classes â†’ features â†’ firmware â†’ required anti-affinity. Tie-break: alphabetical by `(provider_name, fd_name)`. `Decision::to_scheduled_placement(now)` converts to the API type.
- **Backend ID resolution rule (2026-05-26):** when a Provider declares `StorageClassMapping { name: "gold", target: { datastore: "ds-1" } }` (or similar), the scheduler resolves "gold" to a single backend_id string by taking **the first value of the target BTreeMap in lexical key order**. This works for: `{datastore: ds-1}` â†’ "ds-1", `{datastoreCluster: dsc-gold}` â†’ "dsc-gold", `{tagCategory: tier, tag: gold}` â†’ "gold" (because BTreeMap sorts by key, and `tag` < `tagCategory`). If no Provider mapping exists at all the backend_id falls back to the class name itself. The provider's reconciler (Phase 1B+) is the consumer; vSphere takes a single `datastore: String` so this rule produces what it needs.
- **InfraMachineRead trait pattern (2026-05-26):** `crates/banlieue-controller/src/reconciler/status_mirror::InfraMachineRead` accessors (`initialization`, `addresses`, `failure_domain`, `provider_id`, `conditions`) keep the reconciler decoupled from provider-specific types. Phase 1C / 1D add `impl InfraMachineRead for ProxmoxMachine` / `LibvirtMachine` without touching the reconciler. The impl for `VSphereMachine` uses a `static NO_INIT: InitializationStatus` fallback for the pre-status case.
- **VirtualMachine status-mirror aggregate Ready rule:** `Ready=True` iff `Scheduled=True && (PlacementValid != False) && InfrastructureReady=True`. Reasons when False: `Scheduling` (not scheduled), `PlacementInvalid` (current placement doesn't satisfy spec), `InfrastructureNotReady` (infra not ready), `Reconciled` (success). The `PlacementValid` condition is treated as "valid unless explicitly False" so we don't require it to exist before iteration 3 lands the migration loop.
- **Infra CR naming convention (2026-05-26):** the owned `VSphereMachine` shares **name + namespace** with the parent `VirtualMachine`. This is the convention for 1:1 owned infra CRs â€” keeps the relationship discoverable via `kubectl get vspheremachine <vm-name>` without needing an index. Owner-reference back to the VM has `controller: true, blockOwnerDeletion: true`.
- **kube field manager strings are unique per controller (locked):** `banlieue.io/controller` (main controller writes to VMs + InfraMachines), `banlieue.io/provider-vsphere` (vSphere provider writes to VSphereMachine status + VMImage perProvider status + Provider status). Two managers writing the same field cause SSA to fight unless one yields ownership â€” never reuse a field manager string across binaries.
- **Smoke-test boundary at iteration 2 (2026-05-26):** the controller now reconciles end-to-end against the kind cluster but stops at `Scheduled=False reason=ImageNotReady` because no provider populates `VMImage.status.perProvider[].ready=true`. Confirmed working: watch â†’ scheduler â†’ typed-error condition write â†’ requeue. NOT yet confirmed (requires Phase 1B vSphere provider): VSphereMachine SSA, status mirror with a real provider populating fields.
- **Leader election shape (Phase 1A iter 4, 2026-05-26):** rolled in `banlieue-provider-sdk/src/leader.rs` rather than pulling a third-party crate â€” `kube-runtime` 3.1 has no built-in. Pure decision function `decide_action(now: Timestamp, lease: Option<&Lease>, cfg) -> LeaseAction { AcquireNew | Renew | Wait | TakeOver }` is unit-tested without a cluster; async `acquire_or_wait` + `renew_forever` are thin wrappers around `Api<Lease>` GET / CREATE / SSA PATCH. Field manager: `banlieue.io/leader-election`. Boundary semantics: `renew_time + lease_duration == now` is still alive (`<=`, not `<`) to avoid racing the holder's renewal cycle. Defaults match `kube-controller-manager`: 15s lease, 5s renew, 2s retry. Identity sourced from `POD_NAME` â†’ `HOSTNAME` â†’ "unknown".
- **Controller startup sequence (post-iter-4):** parse CLI â†’ init tracing â†’ `build_client` â†’ spawn health server â†’ `acquire_or_wait` (unless `--no-leader-elect`) â†’ spawn `renew_forever` background task â†’ build `Controller` â†’ race controller stream against `shutdown_signal()` (SIGTERM + Ctrl-C) via `tokio::select!`. Renew failure inside the background task calls `std::process::exit(1)` so the Deployment restarts the pod and re-enters leader election cleanly.
- **MicroTime is *not* Copy in k8s-openapi 0.27 even when its inner `jiff::Timestamp` is.** Constructing two `MicroTime` values from one `Timestamp::now()` looks like a Copy move and is rejected; the fix is to hold a `let now = Timestamp::now();` and wrap it twice (`MicroTime(now)`, `MicroTime(now)`) â€” `Timestamp` itself is Copy. Same gotcha will surface when constructing acquire_time/renew_time pairs.
- **TDD test-file `use` path with `#[path]`-included modules:** when `src/foo.rs` declares `#[cfg(test)] #[path = "foo_tests.rs"] mod foo_tests;`, the inner `mod tests` inside `foo_tests.rs` sees `super::super` as `crate::foo` (the parent file's namespace) â€” **not** `crate::foo::foo_tests` or `crate`. So tests import via `use super::super::{Type, fn};`, NOT `use super::super::foo::{...}`. Got this wrong on the leader_tests first cut.

- **[2026-05-26] YAML 1.1 implicit-boolean trap in CRD enum variants.** Go's YAML 1.1 parser (used by the kube apiserver) interprets bare `On`/`Off`/`Yes`/`No`/`True`/`False`/`y`/`n` (any case) as booleans. If a CRD enum variant generates one of those bare tokens in its `enum:` or `default:` keys, the apiserver rejects the CRD with `Invalid value: "boolean": ... must be of type string`. **Fix:** never use these names as variant identifiers for types that end up in a CRD schema. Use `PoweredOn`/`PoweredOff`/`Suspended` (vSphere/CAPI convention) or any other multi-word form. Verified 2026-05-26: PowerState was the only offender; all other enums (Firmware kebab-case, IpamSource camelCase, etc.) are safe.
- **[2026-05-26] `calm template --clear-output-directory` clobbers hand-written files in the output dir.** The original `calm-diagrams` Makefile target passed `--clear-output-directory` against `docs/src/architecture/`, which is the same directory MkDocs serves to readers. Adding a hand-written `index.md` (or any other companion doc) to that directory means it gets silently deleted on every `make calm-diagrams` / `make docs` run. **Fix:** replace the blanket clear with an explicit `rm -f $(CALM_DIAGRAMS_OUT)/{system,flows}.md $(CALM_DIAGRAMS_OUT)/*.hbs` of only the files the CLI is about to (re)produce. Applied to both `calm-diagrams` and the new `calm-docify` targets.

## Decision Log

<!-- Significant technical decisions with rationale. Why X was chosen over Y. -->

- **[2026-05-22] CRDs are code-first via `banlieue-api`.** No hand-written CRD YAML. The `crdgen` binary is the only producer of `deploy/crds/`. Rationale: prevents drift between Rust types and deployed schemas (which silently breaks server-side apply).
- **[2026-05-22] Provider communication is CRD-only.** Explicitly rejected gRPC/HTTP between main controller and providers â€” the K8s API is the bus. Rationale: makes providers reusable as CAPI infra providers, removes a whole class of auth/networking/version-skew problems. See `~/dev/roadmaps/banlieue/00-OVERVIEW.md` (out-of-repo).
- **[2026-05-25] Multi-crate workspace, not single-crate.** Explicitly considered flattening to a single crate like `kube-rs/controller-rs`. Rejected because the roadmap requires 4+ provider crates that ship as independent images. Adopt controller-rs *per-crate conventions* (Error type, telemetry, metrics, fixtures.rs) when each crate lands; don't preload them now.
- **[2026-05-25] Edition 2024, MSRV 1.85.** Bumped from `2021` / `1.80` to match controller-rs. Rationale: gives access to async closures, new capture rules, and stays aligned with the kube-rs reference. Toolchain check is the only cost.
- **[2026-05-25] Roadmap lives outside the repo.** Moved `docs/roadmap/` to `~/dev/roadmaps/banlieue/` and stripped references from in-repo files (`CLAUDE.md`, `.claude/CLAUDE.md`, `.claude/SKILL.md`, `.github/workflows/build.yaml`, `.wolf/cerebrum.md`). Rationale: OSS projects shouldn't ship the maintainer's planning artifacts. ADRs (`docs/adr/`) and design docs (`docs/design/`) **do** stay in-repo â€” those are public-facing technical records. Roadmap docs follow the same numeric-prefix convention out of tree.
